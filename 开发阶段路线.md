### 🚀 完整开发路线图 (基于现有代码的高内聚低耦合方案)

基于你现在的代码（事件总线驱动、分离的 UI 渲染和独立的状态管理），我建议你按照以下 **4 个阶段** 进行开发。这样能保证系统绝对稳定，绝不会改崩之前的代码：

#### Phase 1: 数据基建与存储扩容 (不动 UI，只改数据结构)

- **动作：**
    
    1. 修改 `GlobalState.h`，将 `alarms[3]` 改为 `alarms[8]`，在 `AlarmData` 结构体中新增 `uint8_t repeatMode` (0=单次, 1=每天, 2=工作日)。
        
    2. 在 `GlobalState.h` 中新增统一任务相关的状态变量：`enum TimerMode { TM_IDLE, TM_STOPWATCH, TM_POMODORO, TM_INTERVAL }`。
        
    3. 修改 `WebGateway.cpp` 的 `0x0F` (设置闹钟) 指令解析，多读取一个 byte 的 `repeatMode`。
        
- **目标：** 先让 ESP32 能存下 8 组闹钟和新的配置，确保断电重启后用串口打印能看到数据恢复正常。
    

#### Phase 2: 剥离“统一计时引擎” (The Unified Timer Task)

- **动作：**
    
    1. **新建 `TimerManager.cpp/.h`**。不要把所有计时逻辑都塞在 `main.cpp` 的 `loop()` 里了。
        
    2. 在这个 Manager 里创建一个 FreeRTOS Task（如 `xTaskCreate(TimerTask, ...)`），以 1 秒为周期循环 (`vTaskDelay(1000)` )。
        
    3. 当 Task 发现正计时/番茄钟/间歇时间到了，直接向 `EventBus` 发送自定义事件（比如 `ACT_TIMER_PHASE_END`）。`main.cpp` 的 `handleEvent` 收到事件后，再统一调用 `VoiceAssistant_SendX` 发送播报。
        
- **目标：** 实现切屏不中断。你可以切到时钟界面，后台的番茄钟依然在走，时间到了主循环会自动收到事件并播报。
    

#### Phase 3: 重构 UI 渲染层 (16×8 规范落地)

- **动作：**
    
    1. 在 `DisplayCore.cpp` 中新增针对 4 位数字紧凑显示的工具函数（最大显示 99:59）。
        
    2. 修改 `main.cpp` 中的 `renderTimer`（正计时）、新增 `renderPomodoro` 和 `renderInterval`。在这里实现底部 8 个像素的“蓝红绿”覆盖逻辑。
        
    3. 修改 `renderAlarm`，适配底部的 8 组独立指示灯。
        
- **目标：** 视觉效果完全达到文档要求，所有 UI 渲染与后台逻辑完全解耦。
    

#### Phase 4: 骑行副中心与语音调度中心

- **动作：**
    
    1. **新建 `CyclingManager.cpp`**。将 `VoiceAssistant.cpp` 里散落的心率/踏频越界判断移到这里，并加入严格的“3秒连续越界”防抖判断。
        
    2. 建立“一连四问”的初始化状态。
        
    3. 建立“免打扰拦截器”：在 `main.cpp` 触发闹钟前，加一句 `if (AppState.dndEnabled && AppState.currentMode == MODE_RIDING) return;`。
