# Pixel-Box OS 核心演进 (V30+)


## 一、 系统基石与铁律

- **开发铁律**
    - 高内聚，低耦合
- **单一任务源原则 (The Unified Timer)**
    - 正计时 / 番茄钟 / 间歇训练，在底层共用同一个 FreeRTOS Task (互斥运行)。
    - 跨屏保活：切屏仅改变 UI 映射，后台 Task 永不中断，退回主页依然倒数并按时响铃。
- **全局报警防波动作战**
    - 心率 / 踏频 / 计时结束 / 闹钟，所有异常状态必须连续满足 3 秒才触发报警，杜绝瞬时毛刺。
- **暴力抢占式语音分发**
    - 利用 SU-03T 串口直发覆盖特性，无需检测 Busy 引脚。
    - 绝对优先级：骑行报警 (高优打断) > 骑行节拍 > 计时状态 > 普通闹钟。
- **原生时间中枢**
    - 依赖 Web 端下发的 Unix Timestamp `settimeofday`。
    - ESP32 内部 RTC 原生解析“星期几 (Weekday)”，完美支撑工作日闹钟逻辑，断网亦可准时触发。

## 二、 视效渲染矩阵 (16×8 规范)

- **常规分区模式 (左 8×8 图标 + 右 8×8 字符)**
    - 番茄钟：左番茄图标 + 右侧纯分钟数 (工作红 / 休息绿)。
    - 间歇时钟：左骑行图标 + 右侧纯分钟数 (骑行红 / 休息绿)。
- **沉浸正计时模式 (全屏高分屏)**
    - 核心数值：无图标，直接显示最高 99 分 59 秒的 4 位数字 (MM:SS)。
    - 底部像素打点 (底行 8 个间隔像素位)：
        - 循环色彩覆盖：1-8次 (蓝色) ➔ 9-16次 (红色) ➔ 17-24次 (绿色)，周而复始覆盖最早数据。
- **闹钟矩阵指示**
    - 屏幕底行 8 个像素位，亮起间隔点指示当前查看的闹钟组号（1-8组）。

## 三、 按键与语音交互同构

- **实体 4 键复用 (MODE / + / - / OK)**
    - 交互统一：短按 = 切换 / 操作；长按 = 确认 / 打点 / 清零 / 回主页。
- **语音全量模拟按键**
    - 语音板接收口令后，转化为对应的“按键 Hex 码”下发，ESP32 底层零侵入，完全复用 `EventBus` 事件流。
    - 零侵入开发：ESP32 底层完全复用实体按键的 `EventBus` 事件流。

## 四、 核心业务域：骑行智能辅训

- **1. 骑行初始化 (一连四问)**
    - 触发：按下 OK 进入骑行模式。
    - 流程：系统单条语音连问 (免打扰 / 节拍 / 心率 / 踏频)。
    - 录入：连贯回答以生成配置树。若超时无响应或识别失败，默认降级采用**上一次成功保存的配置**。
- **2. 双轨子模式**
    - 自由骑行：随时切出，后台静默监督。
    - 间歇训练：绑定后台间歇时钟 Task，自动切分阶段，组间语音盲操提醒。
- **3. 骑行免打扰屏障 (DND)**
    - 开启后强行静默 8 路闹钟、番茄钟、非骑行提示音，全屏专注骑行数据。
- **4. 阈值与节拍巡检**
    - Web 下发配置：心率阈值 / 踏频区间 / BPM 节拍 (60-120)。
    - 语音快捷微调：“加快节拍 / 减慢节拍”。

## 五、 存储与 Web 控制台统一

- **Flash 寿命保护**
    - NVS 键名规范化，严格执行 10 秒防抖延迟写入。
- **Web 极简表单**
    - 番茄 / 间歇：工作(骑行)时长、休息时长、循环次数。
    - 闹钟矩阵：8 路一致，新增“重复规则 (单次/每天/工作日)”字段。
    - 骑行配置：心率区间、踏频区间、BPM 节拍。
