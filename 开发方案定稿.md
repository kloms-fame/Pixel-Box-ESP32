# Pixel-Box OS 核心演进 (V30+)


## 开发铁律

- **高内聚，低耦合**
- **在现有基础上进行增加新的功能，不过度影响之前的功能**
- **确保可维护性，可二次开发性**

## 一、 系统基石与铁律

- **单一任务源原则 (The Unified Timer)**
    - 正计时 / 番茄钟 / 间歇训练，在底层共用同一个 FreeRTOS Task。
    - 绝对互斥：启动任意一项，自动清空并覆盖前序任务。
    - 跨屏保活：切屏仅改变 UI 映射，后台 Task 永不中断，退回主页依然按时响铃。
- **全局报警与防波动作战**
    - 心率 / 踏频 / 计时结束 / 闹钟，所有异常状态必须连续满足 3 秒才触发报警，杜绝瞬时毛刺误报。
- **暴力抢占式语音分发 (无 BUSY 机制)**
    - 优先级队列：骑行报警 (高优打断) > 骑行节拍 > 计时状态 > 普通闹钟。
    - 发送策略：利用 SU-03T 串口新指令直接覆盖旧语音的物理特性，无需检测 Busy 引脚。
- **原生时间中枢**
    - 依赖 Web 端下发的 Unix Timestamp `settimeofday`。
    - ESP32 内部 RTC 原生 `localtime_r()` 解析“星期几 (Weekday)”，完美支撑工作日闹钟逻辑，断网亦可正常工作。
- **Web控制界面**
    - 能配置管理复杂内容
    - 能开启关闭功能
    - 能切换界面

## 二、 视效渲染矩阵 (16×8 规范)

- **常规分区模式 (左 8×8 图标 + 右 8×8 字符)**
    - 番茄钟：左番茄图标 + 右侧纯分钟数 (工作红 / 休息绿)。
    - 间歇时钟：左骑行图标 + 右侧纯分钟数 (骑行红 / 休息绿)。
- **沉浸正计时模式 (全屏高分屏)（在现有基础上进行修改优化增加新功能）**
    - 核心数值：无图标，直接显示最高 99 分 59 秒的 4 位数字 (MM:SS)。
    - 底部像素打点 (最底行 16 个像素中挑 8 个点，间隔显示)：
        - 循环覆盖色彩学：1-8次 (蓝色) ➔ 9-16次 (红色) ➔ 17-24次 (绿色)，周而复始。

## 三、 按键与语音交互同构

- **实体 4 键复用 (MODE / + / - / OK)**
    - 交互统一：短按 = 切换 / 操作；长按 = 确认 / 打点 / 清零 / 回主页。
- **语音全量模拟按键**
    - “模式 / 加 / 减 / 确认”指令，由语音板翻译为 Hex 码。
    - 零侵入开发：ESP32 底层完全复用实体按键的 `EventBus` 事件流。
    - 语音板接收口令后，转化为对应的“按键 Hex 码”下发，ESP32 底层零侵入，完全复用 `EventBus` 事件流。

## 四、 核心业务域：骑行智能辅训

- **1. 骑行初始化 (一连四问)**
    - 触发：按下 OK 进入骑行模式。
    - 流程：系统单条长语音发起询问 (免打扰 / 节拍 / 心率 / 踏频)。
    - 录入：用户以类似“定闹钟”的连贯句式回答，确立本次骑行配置。
    - 超时降级：如果超时未回答或识别出错，默认采用**上一次成功保存的骑行配置**。
- **2. 双轨子模式**
    - 自由骑行：随时可切出看其他界面，不影响后台动作。
    - 间歇训练：深度绑定后台间歇时钟 Task，自动切分阶段并提供语音盲操组间提醒。
- **3. 骑行免打扰屏障 (DND)**
    - 拦截器：一旦开启，系统强行静默 8 路闹钟、番茄钟、非骑行类的提示音。
    - 专注模式：全屏锁定骑行数据，屏蔽一切无关切屏请求。
- **4. 阈值与节拍巡检**
    - Web 端统一下发配置：心率高低阈值 / 踏频区间 / BPM 节拍 (60-120 范围)。
    - 支持骑行中语音快捷微调：“加快节拍 / 减慢节拍”。
- **5.记录数据**
    - 记录当日的总骑行里程和骑行时间
    - 在web页面可以设置骑行目标

## 五、 存储与 Web 控制台统一

- **Flash 寿命保护**
    - 存储统一下沉至 NVS，严格执行 20 秒防抖延迟写入。
- **Web 极简表单（新开一页，与之前的隔离）**
    - 番茄 / 间歇：收敛至 3 项核心 (工作/骑行时长、休息时长、循环次数)。
    - 闹钟矩阵：8 路结构完全一致，新增“重复规则 (单次/每天/工作日)”字段。底部的灰色指示点扩展为 8 个。
    - 骑行配置：收敛至心率区间、踏频区间、BPM 节拍设定。
    - 正计时：显示所有的打卡点
    - 页面选项：可以选择切换到任意界面
    - 骑行模式：显示当前骑行时间，骑行踏频，心率，
    - 骑行日目标：设定进入骑行总里程，显示今日累计骑行量和骑行时间
- **专注骑行页面（只有骑行相关内容）（单开一页）**
    - 骑行目标
    - 今日骑行时间
    - 今日骑行里程
    - 总共累计骑行数据（缓存保存到本地数据）

## 六、 语音播报矩阵 (完善补充)

- **骑行播报**
    - 触发报警：“踏频过低/过高”、“心率下降，请加快踏频”、“心率过高，注意负荷”。
    - 节拍反馈：“当前节拍 X，保持节奏”。
    - 初始化反馈：按顺序回答完毕后，播报“配置完成，骑行开始”。
- **计时播报**
    - 正计时打点：“打点成功，第 X 点，用时 X 分 X 秒”。
    - 正计时结束：“正计时完成，总时长 X 分钟”。
    - 番茄钟：“番茄工作/休息结束”；“番茄完成，共 X 组，总时长 X 分钟”。
    - 间歇训练：“间歇休息/骑行开始”；“间歇训练结束，共循环 X 次”。
